Setup Ghost for GitHub Pages (For OS X)

04 December 2015

To build your own static GitHub Pages with Ghost, just follow these steps.

Install Node.js

Ghost is written in Node.js, so you will need the Node.js runtime.

  • Install the package from the .pkg
  • Make sure that /usr/local/bin is in your $PATH
  • Run node -v to check

Setup Ghost locally

Check for the latest version of Ghost from Ghost.org

  • Unzip and go to the directory
  • npm install
  • npm start to run Ghost locally
  • Visit http://127.0.0.1:2368/ghost to create an admin account and setup Ghost locally

Install Buster

With the tool Buster you can export the Ghost blogs into static pages. First we install Buster with:

  • easy_install pip (to install pip)
  • pip install buster(to install buster)
  • brew install wget (required by buster)

Create GitHub Pages repo

  • Go to your GitHub account and create a public repo named accountname.github.io (in my case it was derekcz1991.github.io)

Prepare static folder

  • I switched to a different folder from the Ghost installation to keep the generated static files separately (e.g. /Users/derek/Documents/blog-static/)

Set up Buster

  • Under folder /blog-static run buster setup and enter the repo address you just created when prompted (e.g. https://github.com/derekcz1991/derekcz1991.github.io)
  • Once this is done, run buster generate --domain=http://127.0.0.1:2368 which will generate all the needed files under the static subfolder created in the previous step.
  • cd static to switch into static subfolder, git add . and then git commit -m 'Initial commit' followed by a git push to upload your repo to GitHub.

Visit your Blog

  • Now you can visit your static Ghost blog posts (for me derekcz1991.github.io)

Write new Blog posts

To write new Blog posts or update the existing ones, just start Ghost, edit, then publish and deploy the static pages.

  • cd ghost
  • npm start
  • cd blog-static
  • buster generate --domain=http://127.0.0.1:2368
  • buster deploy
